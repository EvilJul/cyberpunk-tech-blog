API接口设计

公开API和管理API设计

公开API (端口9098)
方法 路径 功能 响应
GET /health 健康检查 { status: "ok", timestamp }
GET /api/articles 获取文章列表 文章数组（分页、筛选）
GET /api/articles/:slug 获取文章详情 完整文章对象
GET /api/search 搜索文章 搜索结果数组
GET /api/articles/:id/comments 获取文章评论 评论数组
POST /api/articles/:id/comments 提交评论 新评论对象
管理API (端口3033)
方法 路径 功能 备注
GET /api/articles 读取文章 用于编辑
POST /api/articles 创建文章 自动生成ID
PUT /api/articles/:id 更新文章 自动备份
DELETE /api/articles/:id 删除文章 软删除
POST /api/upload 上传图片 multipart/form-data
GET /api/comments 管理评论 审核、删除
PUT /api/comments/:id 更新评论状态 批准/拒绝

认证机制

管理后台认证
• 密码认证（环境变量配置）
• localStorage会话存储
• SSH隧道物理隔离
评论防护
• 评论需审核后显示
• 防spam基础验证
• 速率限制
