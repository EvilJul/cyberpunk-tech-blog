数据模型设计

JSON文件存储结构

数据文件布局
项目根/
├── src/data/ ← 源码模板（Git跟踪）
├── articles.json ← 文章初始模板
└── comments.json ← 评论初始模板
└── server/data/ ← 持久化目录（不跟踪，部署不覆盖）
├── articles.json ← 用户实际编辑的文章
├── comments.json ← 用户实际编辑的评论
├── config.json ← 站点配置
└── uploads/ ← 用户上传的图片

articles.json 模型

{
"articles": [{
"id": "article-001",
"title": "文章标题",
"slug": "article-slug",
"content": "文章内容（Markdown格式）",
"excerpt": "文章摘要",
"category": "技术分类",
"tags": ["标签1", "标签2"],
"author": "作者名",
"publishDate": "2024-01-01",
"updateDate": "2024-01-01",
"status": "published",
"views": 0,
"likes": 0
}]
}

comments.json 模型

{
"comments": [{
"id": "comment-001",
"articleId": "article-001",
"parentId": null,
"author": "评论者",
"email": "email@example.com",
"content": "评论内容",
"createdAt": "2024-01-01T10:00:00Z",
"status": "approved"
}]
}

部署时的数据策略

首次运行
• 自动从 src/data/ 迁移到 shared/data
• 创建初始数据文件
• 设置符号链接
后续部署
• 直接符号链接共享目录
• 数据零丢失
• 自动备份机制
