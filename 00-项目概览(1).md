## Skill 1: 项目架构全景图

> [!abstract]- 展开 Skill 1 — 了解整体架构
>
> ### 技术栈一览
>
> | 层级 | 技术 | 版本 |
> |------|------|------|
> | 前端框架 | React | 18.3 |
> | 构建工具 | Vite | 6.x |
> | CSS 框架 | Tailwind CSS | 3.4 |
> | 动画库 | Motion (Framer Motion) | 12.x |
> | 3D/WebGL | Three.js + OGL | 0.184 / 1.0 |
> | 图标库 | Lucide React | 0.460 |
> | 后端框架 | Express | 5.2 |
> | 图片处理 | Sharp | 0.35 |
> | 部署 | Bash + rsync + systemd | — |
>
> ### 架构拓扑
>
> ```
> ┌─────────────────────────────────────────────────────────┐
> │                    用户浏览器                             │
> │  https://example.com/myResume/                     │
> └────────────┬────────────────────────────┬───────────────┘
>              │                            │
>              ▼                            ▼
> ┌──────────────────────┐   ┌──────────────────────────────┐
> │   Nginx (80/443)     │   │  SSH Tunnel (管理后台)        │
> │   /myResume/ → :9098 │   │  -L 3033:127.0.0.1:3033      │
> └──────────┬───────────┘   └──────────────┬───────────────┘
>            │                              │
>            ▼                              ▼
> ┌─────────────────────────────────────────────────────────┐
> │              Express Server (Node.js)                    │
> │  ┌──────────────────┐  ┌──────────────────────────────┐ │
> │  │  publicApp:9098  │  │     adminApp:3033            │ │
> │  │  (0.0.0.0 公开)   │  │  (仅本地/SSH隧道访问)        │ │
> │  │  · 前端 SPA       │  │  · admin.html SPA           │ │
> │  │  · 公开 API (只读) │  │  · 管理 API (读写)          │ │
> │  │  · 静态资源        │  │  · 图片上传                 │ │
> │  └──────────────────┘  └──────────────────────────────┘ │
> │                         │                                │
> │                    ┌────▼─────┐                          │
> │                    │ 数据持久层 │                          │
> │                    │ shared/   │                          │
> │                    │  data/    │                          │
> │                    └──────────┘                          │
> └─────────────────────────────────────────────────────────┘
> ```
>
> ### 核心设计决策
>
> | 决策 | 原因 |
> |------|------|
> | **双端口架构** | 物理隔离公开/管理 API，无需认证中间件 |
> | **单页应用 (SPA)** | 简历场景无需路由，单页滚动体验更好 |
> | **WebGL 三层背景** | PixelSnow → SideRays → GridOverlay → SplashCursor 层层叠加 |
> | **零依赖管理后台** | admin.html 纯 vanilla JS，无框架无构建，Express 直接 serve |
> | **符号链接持久化** | `server/data → shared/data`，部署永不覆盖用户数据 |
> | **VITE_BASE_PATH** | 全链路支持子路径部署，开发/生产环境切换零代码改动 |

> ### 组件设计模式
>
> 1. **数据驱动** — 所有内容从 `resume.json` / `works.json` 读取，组件纯展示
> 2. **ErrorBoundary 包裹** — 所有 WebGL 组件外层包裹 `ErrorBoundary`，防止 WebGL 不可用导致白屏
> 3. **IntersectionObserver** — 非关键组件（PixelSnow, SideRays）不可见时暂停渲染循环
> 4. **CSS-in-JS (内联)** — 动态样式（鼠标位置、随机旋转）使用内联 style，静态样式使用 Tailwind
> 5. **SPA 无路由** — 单页滚动，无 react-router，极简架构

## Skill 4: 后端双端口服务设计

> [!abstract]- 展开 Skill 4 — 了解后端服务
>
> ### 双端口架构
>
> | 实例 | 端口 | 绑定 | 用途 |
> |------|------|------|------|
> | `publicApp` | 9098 (PUBLIC_PORT) | `0.0.0.0` | 前端 SPA + 静态资源 + 公开只读 API |
> | `adminApp` | 3033 (ADMIN_PORT) | `0.0.0.0` | 管理 SPA + 读写 API + 图片上传 |
>
> 安全模型：物理隔离。管理端口不经过 Nginx 暴露，通过 SSH 隧道访问：
> ```bash
> ssh -L 3033:127.0.0.1:3033 deploy@<服务器IP>
> # 浏览器打开 http://127.0.0.1:3033/admin
> ```
>
> ### 公开 API (publicApp)
>
> | 方法 | 路径 | 功能 | 响应 |
> |------|------|------|------|
> | GET | `/health` | 健康检查 | `{ status: "ok", timestamp }` |
> | GET | `/api/resume` | 获取简历(只读) | 完整 resume JSON |
> | GET | `/api/works` | 获取作品(只读) | 完整 works JSON |
> | GET | `/api/works/:id/page` | 作品详情页(SSR HTML) | 完整 HTML 文档 |
> | POST | `/api/works/:id/click` | 记录作品点击 | `{ success, clicks }` |
> | GET | `/api/counter` | 读取访问计数 | `{ count, lastUpdated }` |
> | POST | `/api/counter/increment` | 递增访问计数 | 更新后的 counter 对象 |
>
> ### 管理 API (adminApp)
>
> | 方法 | 路径 | 功能 | 备注 |
> |------|------|------|------|
> | GET | `/health` | 健康检查 | 同上 |
> | GET | `/api/resume` | 读取简历 | 用于编辑 |
> | PUT | `/api/resume` | **覆盖保存简历** | 自动备份 `.bak.<timestamp>` |
> | GET | `/api/works` | 读取作品 | 用于编辑 |
> | PUT | `/api/works` | **覆盖保存作品** | 自动备份 |
> | GET | `/api/config` | 读取配置 | 文件不存在返回 `{}` |
> | PUT | `/api/config` | **覆盖保存配置** | 自动备份 |
> | POST | `/api/upload` | **上传图片** | multipart/form-data，Sharp 压缩 |
> | GET | `/api/resume/html` | **导出简历 PDF/HTML** | 完整 SSR HTML 文档，含打印样式 |
> | GET | `/api/counter` | 读取计数 | 同公开 |
> | POST | `/api/counter/increment` | 递增计数 | 同公开 |
>
> ### 图片上传流程
>
> ```
> 浏览器 FormData
>   │
>   ▼
> POST /api/upload (multipart/form-data)
>   │
>   ▼
> 手动 Buffer 解析 (无 multer 依赖)
>   ├── 提取 boundary → 定位文件数据
>   ├── 提取原始文件名 + 扩展名
>   └── 生成唯一名: <timestamp>-<random6>.<ext>
>   │
>   ▼
> Sharp 压缩
>   ├── 最大宽度: 1200px
>   ├── 格式: JPEG (质量 80)
>   ├── 不放大 (withoutEnlargement)
>   └── 失败 → 回退原始文件
>   │
>   ▼
> 保存到 server/data/uploads/
>   │
>   ▼
> 返回 { success: true, url: "/uploads/<filename>" }
> ```
>
> ### 管理后台 (admin.html)
>
> **纯 Vanilla JS 单文件 SPA**，零依赖，零构建：
>
> - **认证**：密码认证（通过环境变量配置），localStorage 会话
> - **布局**：240px 侧边栏 + 主编辑区
> - **导航**：基本信息 / 专业技能 / 工作经历 / 项目经验 / 教育背景 / 自我评价 / 作品展示
> - **编辑器**：innerHTML 动态渲染表单，`data-*` 属性绑定字段
> - **自动保存**：30 秒检测修改 → 自动 PUT API
> - **修改跟踪**：`isDirty` 标志，保存后状态指示器
> - **导出**：预览网站 / 导出 PDF(调用 `/api/resume/html`)
> - **图片管理**：封面图 + 多图上传预览删除
>
> ### 路径重写中间件
>
> 为支持子路径部署，`publicApp` 注册路径重写：
>
> ```javascript
> // 将 /myResume/api/* 重写为 /api/*
> if (normalizedBasePath) {
>   publicApp.use((req, res, next) => {
>     if (req.path.startsWith(normalizedBasePath + '/api')) {
>       req.url = req.url.replace(normalizedBasePath, '')
>     }
>     next()
>   })
> }
> ```
>
> ### 静态文件配置
>
> ```
> 子路径 (/myResume)
>   ├── dist/          → maxAge=1d, ETag, MIME 修正
>   ├── uploads/       → 用户上传图片
>   └── index.html     → SPA fallback
>
> 根路径 (/)
>   ├── dist/          → 同上（直接 IP 访问回退）
>   └── index.html     → SPA fallback
> ```
>
> CSS/JS 显式设置 Content-Type 避免 MIME 错误：
> ```javascript
> setHeaders: (res, filePath) => {
>   if (filePath.endsWith('.js'))
>     res.setHeader('Content-Type', 'application/javascript; charset=utf-8')
>   if (filePath.endsWith('.css'))
>     res.setHeader('Content-Type', 'text/css; charset=utf-8')
> }
> ```

## Skill 5: 数据模型与持久化

> [!abstract]- 展开 Skill 5 — 了解数据结构
>
> ### 数据文件布局
>
> ```
> 项目根/
> ├── src/data/                    ← 源码模板（Git 跟踪）
> │   ├── resume.json              ← 简历初始模板
> │   └── works.json               ← 作品初始模板
> │
> └── server/data/                 ← 持久化目录（不跟踪，部署不覆盖）
>     ├── resume.json              ← 用户实际编辑的简历
>     ├── works.json               ← 用户实际编辑的作品
>     ├── counter.json             ← 访问计数器
>     ├── config.json              ← 站点配置
>     └── uploads/                 ← 用户上传的图片
> ```
>
> ### 部署时的数据策略
>
> ```
> /opt/myResume/
> ├── shared/data/                 ← 持久化数据（永不删除）
> │   ├── resume.json
> │   ├── works.json
> │   ├── counter.json
> │   ├── config.json
> │   └── uploads/
> │
> ├── releases/
> │   └── 20260616_xxx/
> │       └── server/
> │           └── data → /opt/myResume/shared/data  (符号链接)
> │
> └── current → releases/20260616_xxx/
> ```
>
> **首次运行**：自动从 `src/data/` 迁移到 `shared/data/`
> **后续部署**：直接符号链接共享目录，数据零丢失
>
> ### resume.json 模型
>
> ```json
> {
>   "basic": {
>     "name": "姓名",
>     "nameEn": "NAME",
>     "phone": "xxx-xxxx-xxxx",
>     "email": "email@example.com",

## Skill 6: 部署与运维

> [!abstract]- 展开 Skill 6 — 了解部署架构
>
> ### 部署脚本架构 (`deploy.sh`)
>
> ```
> deploy.sh
> ├── parse_args()        参数解析 (--dry-run, --verbose, --rollback, --config)
> ├── load_config()       加载 deploy.config, 首次运行交互式向导
> ├── preflight_checks()  预检查: SSH / Node / npm / 磁盘 / systemd
> ├── build_local()       npm install + VITE_BASE_PATH=... npm run build
> ├── upload_to_remote()  rsync: dist/ + server/ + src/data/ + package.json
> ├── remote_setup()      npm install + 数据迁移 + 符号链接 + systemd 服务
> ├── health_check()      curl GET /health, 30s 超时
> ├── cleanup_old()       保留最近 KEEP_RELEASES 个版本
> └── log_result()        输出部署结果 + 访问地址
> ```
>
> ### 配置文件 (`deploy.config`)
>
> | 变量 | 值 | 说明 |
> |------|-----|------|
> | `REMOTE_HOST` | `<服务器IP>` | 服务器 IP |
> | `REMOTE_USER` | `deploy` | SSH 用户 |
> | `REMOTE_PORT` | `22` | SSH 端口 |
> | `REMOTE_DIR` | `/opt/myResume` | 部署根目录 |
> | `PUBLIC_PORT` | `9098` | 公开服务端口 |
> | `ADMIN_PORT` | `3033` | 管理后台端口 |
> | `VITE_BASE_PATH` | `/myResume/` | 子路径配置 |
> | `KEEP_RELEASES` | `3` | 保留历史版本数 |
> | `SERVICE_NAME` | `my-resume.service` | systemd 服务名 |
>
> ### systemd 服务配置
>
> ```ini
> [Unit]
> Description=My Resume App
> After=network.target
>
> [Service]
> Type=simple
> User=deploy
> WorkingDirectory=/opt/myResume/current
> ExecStart=/usr/bin/node /opt/myResume/current/server/index.js
> Restart=on-failure
> RestartSec=10
> Environment=NODE_ENV=production
> Environment=PUBLIC_PORT=9098
> Environment=ADMIN_PORT=3033
> Environment=VITE_BASE_PATH=/myResume/
>
> [Install]
> WantedBy=multi-user.target
> ```
>
> ### Nginx 配置
>
> ```nginx
> server {
>     listen 443 ssl http2;
>     server_name example.com;
>
>     ssl_certificate     /etc/letsencrypt/live/example.com/fullchain.pem;
>     ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;
>
>     # MyResume
>     location /myResume/ {
>         proxy_pass http://127.0.0.1:9098;   # 末尾无 / ，保留完整路径
>         proxy_http_version 1.1;
>         proxy_set_header Host $host;
>         proxy_set_header X-Real-IP $remote_addr;
>         proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
>         proxy_set_header X-Forwarded-Proto $scheme;
>     }
>
>     # ClawTalk
>     location /clawtalk/ {
>         proxy_pass http://127.0.0.1:10990/;
>         # ...
>     }
> }
> ```
>
> ### 健康检查
>
> ```bash
> # 脚本自动执行
> curl -s -o /dev/null -w "%{http_code}" \
>   http://${REMOTE_HOST}:${PUBLIC_PORT}/health
> # 期望: 200
> # 超时: 30 秒, 间隔: 2 秒
> ```
>
> ### 回滚机制
>
> ```bash
> # 回滚到上一个版本
> ./deploy.sh --rollback
>
> # 回滚到前第 N 个版本
> ./deploy.sh --rollback 3
>
> # 原理：
> #   1. 列出 /releases/ 目录按时间排序
> #   2. 选择第 N 个
> #   3. ln -sfn <target> current
> #   4. systemctl restart
> ```
>
> ### 常用运维命令
>
> ```bash
> # 查看服务状态
> ssh deploy@<服务器IP> 'systemctl status my-resume.service'
>
> # 实时日志
> ssh deploy@<服务器IP> 'journalctl -u my-resume.service -f'
>
> # 重启服务
> ssh deploy@<服务器IP> 'systemctl restart my-resume.service'
>
> # 查看部署目录
> ssh deploy@<服务器IP> 'ls -la /opt/myResume/'
>
> # 检查数据持久化
> ssh deploy@<服务器IP> 'ls -la /opt/myResume/shared/data/'
>
> # 管理后台 SSH 隧道
> ssh -L 3033:127.0.0.1:3033 deploy@<服务器IP>
> # 浏览器打开 http://127.0.0.1:3033/admin
> ```

## Skill 7: 子路径部署方案

> [!abstract]- 展开 Skill 7 — 了解子路径部署全链路
>
> ### 问题场景
>
> 多应用共享同一域名，通过子路径区分：
> - `https://example.com/clawtalk/` → ClawTalk
> - `https://example.com/myResume/` → MyResume
>
> ### 全链路配置
>
> | 环节 | 配置位置 | 关键参数 |
> |------|----------|----------|
> | 构建时 | `.env.production` | `VITE_BASE_PATH=/myResume/` |
> | 构建时 | `deploy.sh` build_local() | 传递 `VITE_BASE_PATH` 环境变量 |
> | 运行时 | `deploy.sh` systemd service | `Environment=VITE_BASE_PATH=/myResume/` |
> | 反向代理 | Nginx config | `location /myResume/ { proxy_pass http://127.0.0.1:9098; }` |
>
> ### 前后端路径协调
>
> ```
> 用户访问: https://example.com/myResume/
>     │
>     ▼
> Nginx: proxy_pass http://127.0.0.1:9098  (保留 /myResume/ 前缀)
>     │
>     ▼
> Express: GET /myResume/
>     │
>     ├── 静态文件 → dist/ 目录, 带 /myResume/ 前缀服务
>     ├── API 请求  → 路径重写 /myResume/api/* → /api/*
>     └── SPA fallback → index.html
>     │
>     ▼
> 浏览器加载 HTML:
>     <script src="/myResume/assets/index-xxx.js">
>     <link href="/myResume/assets/index-xxx.css">
>     │
>     ▼
> 前端 JS 中的 API 调用:
>     getApiUrl('/api/resume') → '/myResume/api/resume'  ← BASE_URL 自动拼接
>     getAssetUrl('/uploads/img.jpg') → '/myResume/uploads/img.jpg'
> ```
>
> ### 切换到根路径部署
>
> 只需修改一个地方：
> ```bash
> # deploy.config 中修改
> VITE_BASE_PATH="/"
> # 然后重新部署
> ./deploy.sh
> ```
>
> ### 切换到其他子路径
>
> ```bash
> # deploy.config 中修改
> VITE_BASE_PATH="/resume/"
> # 同时修改 Nginx:
> # location /resume/ { proxy_pass http://127.0.0.1:9098; }
> # 重新部署
> ./deploy.sh
> ```
>
> ### 开发环境
>
> 本地开发不需要子路径：
> ```bash
> npm run dev      # VITE_BASE_PATH 默认为 /, 访问 http://localhost:9099
> ```
